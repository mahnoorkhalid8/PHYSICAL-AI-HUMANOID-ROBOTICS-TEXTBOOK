# URDF Robot Description for Humanoids

Defining humanoid robot structures using URDF.

## Unified Robot Description Format (URDF)

URDF (Unified Robot Description Format) is an XML format used in ROS to describe robot models. It defines the physical and visual properties of a robot, including links, joints, and their relationships. URDF is fundamental to robot simulation, visualization, and control in ROS-based systems.

## Key Components of URDF

### Links
Links represent rigid bodies in the robot. Each link contains:

- **Visual**: How the link appears in visualization tools
- **Collision**: Collision properties for physics simulation
- **Inertial**: Mass, center of mass, and inertia tensor for dynamics

### Joints
Joints connect links and define their motion constraints. Joint types include:
- `revolute`: Rotational joint with limits
- `continuous`: Rotational joint without limits
- `prismatic`: Linear sliding joint
- `fixed`: No motion between links
- `floating`: 6DOF motion (for base of floating robots)
- `planar`: Motion on a plane

## Xacro for Complex URDFs

For more complex humanoid robots, Xacro (XML Macros) helps manage large URDF files:

```xml
<?xml version="1.0"?>
<robot xmlns:xacro="http://www.ros.org/wiki/xacro" name="humanoid_xacro">

  <!-- Define properties -->
  <xacro:property name="M_PI" value="3.1415926535897931" />
  <xacro:property name="base_mass" value="10.0" />
  <xacro:property name="link_radius" value="0.05" />

  <!-- Macro for a simple leg -->
  <xacro:macro name="leg" params="prefix reflect">
    <link name="${prefix}_hip_link">
      <visual>
        <geometry>
          <cylinder length="0.15" radius="${link_radius}"/>
        </geometry>
        <origin xyz="0 ${reflect*0.08} 0.15" rpy="${M_PI/2} 0 0"/>
      </visual>
      <collision>
        <geometry>
          <cylinder length="0.15" radius="${link_radius}"/>
        </geometry>
        <origin xyz="0 ${reflect*0.08} 0.15" rpy="${M_PI/2} 0 0"/>
      </collision>
      <inertial>
        <mass value="0.8"/>
        <inertia ixx="0.002" ixy="0.0" ixz="0.0" iyy="0.002" iyz="0.0" izz="0.001"/>
      </inertial>
    </link>

    <!-- Additional leg links would follow the same pattern -->
  </xacro:macro>

  <!-- Use the macro to create legs -->
  <xacro:leg prefix="left" reflect="1" />
  <xacro:leg prefix="right" reflect="-1" />
</robot>
```

## URDF in Simulation and Control

URDF files are used by various ROS tools:
- **Gazebo**: For physics simulation
- **RViz**: For visualization
- **MoveIt**: For motion planning
- **ros2_control**: For hardware control

## Best Practices for Humanoid URDFs

1. **Use proper inertial values**: Accurate inertial properties are crucial for stable simulation
2. **Define appropriate joint limits**: Prevent damage to real hardware
3. **Include transmission elements**: For proper hardware control
4. **Use consistent naming**: Follow ROS conventions for joint/link names
5. **Validate URDF**: Use tools like `check_urdf` to verify syntax
